========================================
케어라벨회사 AI Agent 로직 명세
========================================
작성일: 2026-05-21

이 파일을 읽는 AI에게:
이 파일은 케어라벨회사 AI Agent가 수행해야 할 판단 로직을 정의한다.
생산규칙.txt와 라벨코드_규칙.txt의 수치를 기반으로 아래 로직을 구현할 것.
UI는 agent표준 구조/README.md 참고.

※ 케어라벨사는 라벨코드를 직접 생산하므로
  플랫폼 전체의 핵심 키(라벨코드)를 발행하는 회사임.
  출고 데이터에 항상 라벨코드 전체를 포함해야 함.

========================================
1. 주문 수신 시 자동 실행 로직
========================================

주문 데이터 형식:
  - 라벨코드 (9자리) → 케어라벨사가 직접 제작
  - 주문량 (장)
  - 납기일

STEP 1. 라벨코드 유효성 검증
  9자리 형식 확인
  각 자리 코드 유효값 확인 (라벨코드_규칙.txt 참고)
  유효하지 않으면 → 오류 알림, 생산 보류

STEP 2. 소요시간 계산
  소요시간(h) = 주문량(장) ÷ (인쇄기 6대 × 800장/h)
  예) 7,200장 주문 → 7,200 ÷ 4,800 = 1.5시간

STEP 3. 납기 가능일수 계산
  납기 가능일수 = 소요시간 ÷ 9시간 (소수점 올림)

STEP 4. 납기 판정
  오늘 기준 남은 일수 = 납기일 - 오늘
  ✅ 납기 가능: 남은 일수 >= 납기 가능일수
  ⚠ 납기 위험: 남은 일수 < 납기 가능일수 + 1
  ❌ 납기 불가: 남은 일수 < 납기 가능일수

STEP 5. 필요 원자재 계산
  필요 라벨원단(m) = 주문량(장) ÷ 25 (소수점 올림)
  필요 잉크(통)    = 주문량(장) ÷ 10,000 (소수점 올림)

STEP 6. 재고 확인
  라벨원단 재고 >= 필요량 AND 잉크 재고 >= 필요량 → ✅ 즉시 생산 가능
  둘 중 하나라도 부족 → ⚠ 원자재 부족 알림 + 발주 필요량 계산


========================================
2. 재고 모니터링 로직 (상시 실행)
========================================

안전 재고 기준:
  라벨 원단: 500 m
  잉크:       5 통

현재 재고 <= 안전 재고 → AI Agent 패널에 발주 권고 표시
현재 재고 = 0           → ❌ 긴급 발주 알림


========================================
3. 우선순위 결정 로직 (복수 주문 시)
========================================

여러 주문이 동시에 있을 경우:
  1순위: 납기 위험 주문 (남은 일수 적은 순)
  2순위: 동일 납기일 시 → 주문량 적은 것 먼저
  3순위: 동일 조건 시 → 라벨코드 입력 순서대로

AI Agent 패널에 우선순위 순서대로 표시.


========================================
4. 완료 버튼 클릭 시 실행 로직
========================================

작업자가 완료 버튼 클릭 시:
  1. 해당 주문 상태 → "출고완료"로 변경
  2. 재고에서 사용량 차감
     라벨원단: 사용량(m) = 출고량(장) ÷ 25
     잉크:     사용량(통) = 출고량(장) ÷ 10,000
  3. 출고 데이터 플랫폼으로 전송:
     {
       라벨코드: "W3MJW01NV",   ← 핵심 키
       출고량: 장,
       출고일: 날짜,
       회사구분: "케어라벨사",
       파싱정보: {
         브랜드: "W",
         계절: "3",
         성별: "M",
         품목: "J",
         원단: "W",
         스타일: "01",
         컬러: "NV"
       }
     }
  4. 플랫폼이 이 데이터로 옷감사·지퍼단추사 출고 현황과 연결


========================================
5. AI Agent 패널 표시 규칙
========================================

항상 표시:
  - 오늘 기준 진행 중인 라벨코드별 주문 목록
  - 각 주문별 납기 D-? 및 납기 가능 여부
  - 원자재(라벨원단/잉크) 재고 부족 경고

지시사항 표시 우선순위:
  ❌ 라벨코드 오류 → 최상단 오류 표시
  ❌ 납기 불가 주문 → 긴급 표시
  ⚠ 납기 위험 주문 → 우선 착수 권고
  ⚠ 원자재 부족   → 발주 권고
  ✅ 정상          → 오늘의 인쇄 스케줄 안내


========================================
6. 플랫폼 연동 데이터 전송 시점
========================================

  - 주문 수신 시: 납기 가능 여부 + 라벨코드 파싱 정보 전송
  - 출고 완료 시: 라벨코드 포함 출고 데이터 전송 (가장 중요)
  - 재고 부족 시: 재고 부족 알림 전송

  ※ 케어라벨사 출고 완료 신호가 오면
    플랫폼이 옷감사·지퍼단추사에 동일 라벨코드 출고 현황 확인 시작
