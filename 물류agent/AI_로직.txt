========================================
물류회사 AI Agent 로직 명세
========================================
작성일: 2026-05-21

이 파일을 읽는 AI에게:
물류 AI Agent의 핵심은 두 가지다.
1. 납기일 기준 최적 픽업일 자동 계산
2. 왕복 동선 최적화 (빈차 방지)
GPT API를 사용하여 구현.
DB 구조는 DB세션/DB_설계.txt 참고.

========================================
1. 배차 트리거 조건
========================================

플랫폼으로부터 아래 신호 수신 시 자동 배차 시작:
  - 생산사 3곳 중 1곳 이상 출고완료 신호 수신
  - 납기일 D-3 이하로 임박한 화물 감지

배차에 필요한 정보 (platform DB에서 자동 조회):
  - 출발지: 회사정보 테이블 → 시/구 주소
  - 도착지: 인천항 / 부산항 (고정값)
  - 납기일
  - 화물 무게
  - 물품명/수량


========================================
2. 픽업일 자동 계산 로직
========================================

이동시간 기준:
  출발지 → 인천항: 1일
  출발지 → 부산항: 출발지가 부산이면 0.5일, 서울이면 1일

픽업일 계산:
  픽업일 = 납기일 - 이동시간 - 여유 1일

예시:
  납기일 6/10, 부산 출발 → 인천항
  픽업일 = 6/10 - 1일 - 1일 = 6/8

픽업일이 오늘 이전이면:
  → ❌ 납기 불가 긴급 알림


========================================
3. 왕복 최적화 로직 (핵심)
========================================

배차 확정 후 귀환 경로에 추가 화물 연결:

STEP 1. 배차 경로 파악
  예) 부산 사하구 → 인천항

STEP 2. 귀환 경로 확인
  인천항 → 부산 방향

STEP 3. 귀환 경로 내 픽업 가능 화물 검색
  - 도착지가 부산항인 화물 중
  - 픽업 위치가 인천/서울 근방인 화물
  - 기사 귀환 예정일과 납기일 맞는 것

STEP 4. 연결 가능하면 → 왕복 배차 확정
  화물관리 테이블 빈차일지 컬럼: "연결완료" 표시
  연결 불가하면: "빈차 귀환" 표시

예시:
  김기사: 부산 사하구 → 인천항 (6/8 출발)
  ↓ 귀환 시
  인천 근방 픽업 건 있음 → 서울 강남구 픽업 → 부산항 납품
  → 왕복 모두 화물 있음, 빈차 0


========================================
4. 기사 배차 선정 로직
========================================

복수 기사 중 최적 기사 선정 순서:

  1순위: 현재 위치가 픽업지와 같은 시(市)인 기사
  2순위: 가용 상태인 기사 (운행중/휴무 제외)
  3순위: 차량 적재량이 화물 무게 이상인 기사
  4순위: 왕복 최적화 연결 가능한 경로의 기사

모두 동일 조건이면 → 등록 순서대로


========================================
5. 완료 버튼 클릭 시 실행 로직
========================================

기사가 배송 완료 버튼 클릭 시:
  1. delivery 테이블 status → "완료"
  2. complete_date 기록
  3. 기사 status → "가용"으로 자동 전환
  4. 플랫폼 dispatch 테이블 status → "완료"
  5. 왕복 연결 화물 있으면 → 다음 픽업 알림 자동 발송


========================================
6. 하단 AI 지시 영역 표시 규칙
========================================

항상 표시 (우선순위 순):
  ❌ 납기 불가 화물    → 긴급 빨간색 알림
  ⚠  납기 D-1 화물    → 즉시 배차 필요 알림
  ▶  오늘 배차 스케줄  → 기사별 픽업/도착 요약
  🔄 왕복 연결 가능    → 귀환 시 픽업 가능 화물 안내
  ✅ 정상 운행 중      → 현재 운행 현황 요약


========================================
7. 플랫폼 연동
========================================

수신 (플랫폼 → 물류):
  - 생산사 출고완료 신호
  - 납기일/화물정보/무게/도착지

송신 (물류 → 플랫폼):
  - 배차 확정 정보 (기사/차량/픽업일)
  - 배송 완료 신호
  - 왕복 최적화 결과
