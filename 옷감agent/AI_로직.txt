옷감회사 AI Agent 로직 명세
========================================
작성일: 2026-05-21

이 파일을 읽는 AI에게:
이 파일은 옷감회사 AI Agent가 수행해야 할 판단 로직을 정의한다.
생산규칙.txt의 수치를 기반으로 아래 로직을 구현할 것.
UI는 agent표준 구조/README.md 참고.

========================================
1. 주문 수신 시 자동 실행 로직
========================================

주문 데이터 형식:
  - 라벨코드 (9자리)
  - 주문량 (야드)
  - 납기일

STEP 1. 라벨코드 파싱
  5번째 자리 추출 → 원단코드 확인
  예) W3MJW01NV → 5번째 자리 = W → 울(Wool) 원단

STEP 2. 소요시간 계산
  소요시간(h) = 주문량 ÷ (직조기 5대 × 원단별 시간당 생산량)

  원단별 시간당 생산량:
    C(면)        8 야드/h
    P(폴리에스터) 15 야드/h
    L(린넨)       5 야드/h
    W(울)         4 야드/h
    M(혼방)      10 야드/h

STEP 3. 납기 가능일수 계산
  납기 가능일수 = 소요시간 ÷ 9 (일일 가동시간)
  소수점 올림 처리 (0.1일도 하루 필요)

STEP 4. 납기 판정
  오늘 기준 남은 일수 = 납기일 - 오늘
  ✅ 납기 가능: 남은 일수 >= 납기 가능일수
  ⚠ 납기 위험: 남은 일수 < 납기 가능일수 + 1
  ❌ 납기 불가: 남은 일수 < 납기 가능일수

STEP 5. 필요 원자재 계산
  원단별 변환비율:
    C(면)         원사 1kg → 3.0 야드
    P(폴리에스터) 원사 1kg → 5.0 야드
    L(린넨)       원사 1kg → 2.5 야드
    W(울)         원사 1kg → 2.0 야드
    M(혼방)       원사 1kg → 3.5 야드

  필요 원사(kg) = 주문량 ÷ 변환비율 (소수점 올림)

STEP 6. 재고 확인
  현재 재고 >= 주문량 → ✅ 즉시 생산 가능
  현재 재고 < 주문량  → ⚠ 원자재 부족 → 발주 필요량 계산 후 알림


========================================
2. 재고 모니터링 로직 (상시 실행)
========================================

안전 재고 기준:
  C(면)        500 야드
  P(폴리에스터) 300 야드
  L(린넨)       200 야드
  W(울)         150 야드
  M(혼방)       250 야드

현재 재고 <= 안전 재고 → AI Agent 패널에 발주 권고 표시
현재 재고 = 0          → ❌ 긴급 발주 알림


========================================
3. 우선순위 결정 로직 (복수 주문 시)
========================================

여러 주문이 동시에 있을 경우 아래 순서로 우선순위 결정:
  1순위: 납기 위험 주문 (남은 일수 적은 순)
  2순위: 동일 납기일 시 → 주문량 적은 것 먼저 (빠른 완료)
  3순위: 원단 종류 → 느린 원단(W, L) 먼저 착수

AI Agent 패널에 우선순위 순서대로 표시.


========================================
4. 완료 버튼 클릭 시 실행 로직
========================================

작업자가 완료 버튼 클릭 시:
  1. 해당 주문 상태 → "출고완료"로 변경
  2. 재고에서 출고량 차감
  3. 출고 데이터 플랫폼으로 전송:
     {
       라벨코드: "W3MJW01NV",
       원단코드: "W",
       컬러코드: "NV",
       출고량: 야드,
       출고일: 날짜,
       회사구분: "옷감사"
     }
  4. AI Agent 패널 납기 현황 갱신


========================================
5. AI Agent 패널 표시 규칙
========================================

항상 표시:
  - 오늘 기준 진행 중인 주문 목록
  - 각 주문별 납기 D-? 및 납기 가능 여부
  - 재고 부족 경고

지시사항 표시 우선순위:
  ❌ 납기 불가 주문 → 최상단 긴급 표시
  ⚠ 납기 위험 주문 → 우선 착수 권고
  ⚠ 재고 부족     → 발주 권고
  ✅ 정상          → 오늘의 생산 스케줄 안내


========================================
6. 플랫폼 연동 데이터 전송 시점
========================================

  - 주문 수신 시: 납기 가능 여부 플랫폼에 전송
  - 출고 완료 시: 출고 데이터 플랫폼에 전송
  - 재고 부족 시: 재고 부족 알림 플랫폼에 전송
