# 📦 AI-Driven B2B Supply Chain Platform
> **"Data Sovereignty, Collective Intelligence"**
> 분산 에이전트 기반의 의류 부자재 공급망 자동화 및 데이터 역추론 인사이트 플랫폼

## 1. 프로젝트 개요 (Overview)
본 프로젝트는 공급망 내 기업들에게 **표준화된 AI 업무 에이전트**를 배포하여 생산 자동화를 지원하고, 여기서 발생하는 메타데이터를 플랫폼이 수집하여 물류 매칭 및 트렌드 예측 서비스를 제공하는 **B2B 오케스트레이션 솔루션**입니다.

## 2. 핵심 기술 스택
- **Backend:** Python FastAPI (Async, GPT API 연동)
- **Frontend:** React (Vite, Tailwind CSS)
- **Database:** MySQL (기업별 독립 스키마 운영 - 데이터 주권 보호)
- **AI Orchestration:** GPT-4o 기반 멀티 에이전트 통신 및 분석

## 3. 차별화된 아키텍처 설계
### 3.1 비침습적 데이터 수집 (Non-Invasive Data Harvesting)
- **데이터 주권 보장:** 플랫폼은 고객사의 DB에 직접 접근하지 않습니다. 대신 배포된 에이전트가 현장에서 업무를 자동화하고, 판단 결과만 플랫폼에 보고(Push)하는 구조를 채택했습니다.
- **간접적 인사이트 도출:** 라벨 코드(9자리) 파싱 및 물류 요청 데이터를 조합하여 실시간 생산 품목, 원단 선호도, 브랜드별 가동률을 플랫폼이 역으로 추론(Inference)합니다.

### 3.2 AI 에이전트 오케스트레이션
- **현장형 AI 에이전트:** 생산사는 배포된 프로그램을 통해 BL 파싱, 패킹리스트 생성, 기계 배정 등 '귀찮은 사무작업'을 자동화합니다.
- **지능형 리포팅 채널:** 에이전트 AI가 현장 상황(재고 부족, 납기 지연 등)을 요약하여 플랫폼 AI에게 보고하면, 플랫폼 AI는 이를 바탕으로 물류 에이전트와 실시간 배차 매칭을 수행합니다.

## 4. 핵심 가치 및 도메인 로직
### 4.1 라벨 코드를 활용한 트렌드 예측
- 라벨 코드에 포함된 성별/품목/원단 정보를 분석하여 다음 시즌 유행할 컬러와 소재를 예측하는 '데이터 마이닝' 로직을 구현했습니다. (예: "W(울) 원단 및 NV(네이비) 라벨 생산량 급증 -> 가을/겨울 클래식 라인 강세 예측")

### 4.2 물류 매칭 및 배차 최적화
- 생산사의 물동량 정보가 플랫폼에 들어오는 즉시 물류사의 차량 현황과 매칭하여 공차율을 줄이고 물류 비용을 최적화합니다.

## 5. 기술적 문제 해결 (Problem Solving)
### 5.1 분산 환경의 데이터 일관성 해결
  - 5개 독립 DB 구조에서 발생할 수 있는 데이터 유실을 방지하기 위해, 전송 실패 시 롤백 및 재시도 메커니즘을 적용하여 시스템의 안정성을 확보했습니다.

### 5.2 멀티 에이전트 간 '보고(Reporting)' UI 설계
  - 자유 채팅이 아닌 '비즈니스 이벤트 타임라인' UI를 통해 AI가 무엇을 판단하고 플랫폼과 어떻게 소통하는지 가시화했습니다.

## 6. 프로젝트 결과 및 회고
- **B2B SaaS 모델의 기술적 실현:** 기업의 보안 욕구(독립 DB)와 플랫폼의 데이터 욕구(인사이트 수집) 사이의 균형을 맞춘 아키텍처를 설계했습니다.
- **AI 협업의 실제:** AI가 단순 챗봇을 넘어 실제 비즈니스 프로세스를 오케스트레이션(Orchestration)하는 핵심 엔진으로 동작하도록 구현했습니다.
