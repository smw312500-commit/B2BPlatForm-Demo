이 순서대로 파일을 읽고 개발을 시작할 것.

========================================
필독 순서
========================================

1. ../../기획안.txt             전체 플랫폼 구조 이해
2. ../../기술스택.txt           사용 기술 확인 (React + FastAPI + MySQL)
3. ../../CHANGES.md             현재까지 완료된 작업 확인
4. ../../agent표준 구조/README.md  UI 레이아웃 표준 확인
5. 옷감코드_규칙.txt            원단/컬러 코드 규칙
6. 생산규칙.txt                 기계/생산속도/변환비율
7. AI_로직.txt                  AI Agent 판단 로직

========================================
이 세션에서 만들 것
========================================

옷감회사 업무자동화 시스템 (독립 앱)

  frontend/   React
    - 탭 4개: 재고 / 발주하기 / 발주취소 / 기타
    - 우측 AI Agent 패널 (고정)
    - UI 표준: agent표준 구조/README.md 참고
    - 스케치: agent표준 구조/*.jpg 참고

  backend/    FastAPI (Python)
    - 재고 조회/수정 API
    - 발주 등록/취소 API
    - 출고 등록 API (완료 버튼)
    - AI Agent 판단 API (생산스케줄, 납기계산)
    - 플랫폼으로 출고완료 신호 전송 API

  DB: MySQL - company_fabric 스키마
    - fabric_stock   (재고)
    - fabric_order   (발주)
    - fabric_release (출고)
    - 상세: ../../DB세션/DB_설계.txt 참고

========================================
완료 버튼 동작 (핵심)
========================================

작업자가 완료 버튼 클릭
  → FastAPI: fabric_release 테이블 status = "출고완료"
  → FastAPI: fabric_stock 재고 차감
  → FastAPI: 플랫폼 API로 출고완료 신호 전송
  → React: AI Agent 패널 갱신

========================================
AI Agent 패널 데이터 (우측 고정)
========================================

FastAPI /agent/status 엔드포인트:
  - 진행 중인 주문별 납기 D-? 계산
  - 납기 가능 여부 (AI_로직.txt 로직 적용)
  - 재고 부족 여부 (생산규칙.txt 안전재고 기준)
  - 우선순위 지시사항

========================================
개발 시 주의사항
========================================

  - UI 구조는 agent표준 구조/README.md에서 절대 벗어나지 말 것
  - 3개 생산사 UI가 동일 구조여야 함 (컴포넌트 재사용 설계)
  - 출고완료 신호는 반드시 플랫폼으로 전송할 것
  - 원단코드: C/P/L/W/M (옷감코드_규칙.txt 참고)
