========================================
이 세션에서 만들 것
========================================

케어라벨회사 업무자동화 시스템 (독립 앱)

※ 케어라벨사는 플랫폼 전체의 핵심 키(라벨코드)를 발행하는 회사.
  출고 데이터에 반드시 라벨코드 9자리를 포함해야 함.

  frontend/   React
    - 탭 4개: 재고 / 발주하기 / 발주취소 / 기타
    - 우측 AI Agent 패널 (고정)
    - UI 표준: agent표준 구조/README.md 참고

  backend/    FastAPI (Python)
    - 재고 조회/수정 API
    - 발주 등록/취소 API
    - 출고 등록 API (완료 버튼)
    - 라벨코드 유효성 검증 API
    - AI Agent 판단 API
    - 플랫폼으로 출고완료 신호 전송 API (라벨코드 포함)

  DB: MySQL - company_label 스키마
    - label_stock   (재고)
    - label_order   (발주)
    - label_release (출고 - 라벨코드 포함)
    - 상세: ../../DB세션/DB_설계.txt 참고

========================================
완료 버튼 동작 (핵심)
========================================

작업자가 완료 버튼 클릭
  → FastAPI: label_release 테이블 status = "출고완료"
  → FastAPI: label_stock 재고 차감 (라벨원단/잉크)
  → FastAPI: 플랫폼 API로 출고완료 신호 전송
             { 라벨코드: "W3MJW01NV", 출고량: N, 출고일: date }
  → React: AI Agent 패널 갱신

========================================
라벨코드 물품명 규칙
========================================

이 회사의 물품명 = 라벨코드 자체 (9자리)
예) W3MJW01NV

플랫폼 collected_release 테이블:
  item_name = 라벨코드 (예: W3MJW01NV)

라벨코드가 플랫폼에서 옷감사·지퍼단추사와 연결되는 핵심 키임.
