========================================
지퍼단추회사 AI Agent 로직 명세
========================================
작성일: 2026-05-21

이 파일을 읽는 AI에게:
이 파일은 지퍼단추회사 AI Agent가 수행해야 할 판단 로직을 정의한다.
생산규칙.txt의 수치를 기반으로 아래 로직을 구현할 것.
UI는 agent표준 구조/README.md 참고.

※ 라벨코드 4번째 자리(품목코드)로 필요 부자재를 자동 판단하는 것이 핵심.

========================================
1. 주문 수신 시 자동 실행 로직
========================================

주문 데이터 형식:
  - 라벨코드 (9자리)
  - 주문량 (개)
  - 납기일

STEP 1. 라벨코드 파싱 → 필요 부자재 판단
  4번째 자리(품목코드) 추출:

  품목코드  의류품목  필요 부자재
  ────────────────────────────────────────
  T         티셔츠    플라스틱단추
  P         바지      금속단추 또는 지퍼 (기본: 금속단추)
  J         재킷      지퍼 + 금속단추
  D         다운      지퍼

  ※ J(재킷)는 지퍼와 금속단추 둘 다 필요 → 각각 별도 계산

STEP 2. 품목별 소요시간 계산
  소요시간(h) = 주문량(개) ÷ (기계 2대 × 시간당 생산량)

  품목별 시간당 생산량:
    원목단추      20 개/h/대
    플라스틱단추  300 개/h/대
    금속단추      150 개/h/대
    지퍼          200 개/h/대

  예) J(재킷) 200개 주문 시:
    지퍼:    200 ÷ (2 × 200) = 0.5시간
    금속단추: 200 ÷ (2 × 150) = 0.67시간
    총 소요:  두 작업 순차 진행 → 합산

STEP 3. 납기 가능일수 계산
  납기 가능일수 = 총 소요시간 ÷ 9시간 (소수점 올림)

STEP 4. 납기 판정
  오늘 기준 남은 일수 = 납기일 - 오늘
  ✅ 납기 가능: 남은 일수 >= 납기 가능일수
  ⚠ 납기 위험: 남은 일수 < 납기 가능일수 + 1
  ❌ 납기 불가: 남은 일수 < 납기 가능일수

STEP 5. 필요 원자재 계산
  원자재 변환비율:
    원목단추      원목 1kg → 50개
    플라스틱단추  플라스틱 1kg → 200개
    금속단추      금속 1kg → 150개
    지퍼          지퍼테이프 1m → 1개

  필요 원자재 = 주문량 ÷ 변환비율 (소수점 올림)

STEP 6. 재고 확인
  현재 재고 >= 주문량 → ✅ 즉시 생산 가능
  현재 재고 < 주문량  → ⚠ 원자재 부족 → 발주 필요량 계산 후 알림


========================================
2. 재고 모니터링 로직 (상시 실행)
========================================

안전 재고 기준:
  원목:          50 kg
  플라스틱 원료: 100 kg
  금속 원료:     80 kg
  지퍼 테이프:   200 m

현재 재고 <= 안전 재고 → AI Agent 패널에 발주 권고 표시
현재 재고 = 0           → ❌ 긴급 발주 알림


========================================
3. 트렌드 추론 로직 (플랫폼 전송용)
========================================

아래 패턴 감지 시 플랫폼에 트렌드 신호 전송:

  원목단추 출고 증가 (전월 대비 +20% 이상)
  → 신호: "프리미엄 셔츠 라인 수요 증가"

  지퍼 + D(다운) 라벨 출고 급증
  → 신호: "아웃도어 다운 시즌 수요 증가"

  J(재킷) + 금속단추 출고 증가
  → 신호: "포멀 재킷 라인 수요 증가"

  플라스틱단추 출고 급감
  → 신호: "캐주얼 라인 수요 감소"

감지 주기: 월 1회 자동 집계 후 플랫폼 전송


========================================
4. 우선순위 결정 로직 (복수 주문 시)
========================================

여러 주문이 동시에 있을 경우:
  1순위: 납기 위험 주문 (남은 일수 적은 순)
  2순위: 원목단추 주문 우선 (생산 가장 느림)
  3순위: 동일 조건 시 → 주문량 적은 것 먼저

AI Agent 패널에 우선순위 순서대로 표시.


========================================
5. 완료 버튼 클릭 시 실행 로직
========================================

작업자가 완료 버튼 클릭 시:
  1. 해당 주문 상태 → "출고완료"로 변경
  2. 재고에서 출고량 차감
  3. 원자재 재고에서 사용량 차감
  4. 출고 데이터 플랫폼으로 전송:
     {
       라벨코드: "W3MJW01NV",
       품목코드: "J",
       부자재종류: ["지퍼", "금속단추"],
       출고량: { 지퍼: 개수, 금속단추: 개수 },
       출고일: 날짜,
       회사구분: "지퍼단추사",
       트렌드신호: "포멀 재킷 수요 증가" (해당 시)
     }
  5. AI Agent 패널 납기 현황 갱신


========================================
6. AI Agent 패널 표시 규칙
========================================

항상 표시:
  - 오늘 기준 진행 중인 주문 목록 (라벨코드 + 품목코드)
  - 각 주문별 납기 D-? 및 납기 가능 여부
  - 품목별 원자재 재고 현황
  - 재고 부족 경고

지시사항 표시 우선순위:
  ❌ 납기 불가 주문  → 긴급 표시
  ⚠ 납기 위험 주문  → 원목단추 우선 착수 권고
  ⚠ 원자재 부족     → 품목별 발주 권고
  ✅ 정상            → 오늘의 생산 스케줄 안내


========================================
7. 플랫폼 연동 데이터 전송 시점
========================================

  - 주문 수신 시: 납기 가능 여부 전송
  - 출고 완료 시: 출고 데이터 전송
  - 재고 부족 시: 재고 부족 알림 전송
  - 월 집계 시:   트렌드 신호 전송
