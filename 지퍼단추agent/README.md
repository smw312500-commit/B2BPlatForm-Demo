========================================
이 세션에서 만들 것
========================================

지퍼단추회사 업무자동화 시스템 (독립 앱)

  frontend/   React
    - 탭 4개: 재고 / 발주하기 / 발주취소 / 기타
    - 우측 AI Agent 패널 (고정)
    - UI 표준: agent표준 구조/README.md 참고

  backend/    FastAPI (Python)
    - 재고 조회/수정 API
    - 발주 등록/취소 API
    - 출고 등록 API (완료 버튼)
    - AI Agent 판단 API (품목별 납기계산)
    - 플랫폼으로 출고완료 신호 전송 API

  DB: MySQL - company_zipper 스키마
    - zipper_stock   (재고)
    - zipper_order   (발주)
    - zipper_release (출고)
    - 상세: ../../DB세션/DB_설계.txt 참고

========================================
완료 버튼 동작 (핵심)
========================================

작업자가 완료 버튼 클릭
  → FastAPI: zipper_release 테이블 status = "출고완료"
  → FastAPI: zipper_stock 재고 차감
  → FastAPI: 플랫폼 API로 출고완료 신호 전송
             { item_name: "WOOD_BR", 수량: N, 출고일: date }
  → React: AI Agent 패널 갱신

========================================
품목 코드 규칙
========================================

단추: 소재코드_컬러코드
  WOOD_BR     원목단추 브라운
  PLASTIC_BK  플라스틱단추 블랙
  METAL_SV    금속단추 실버
  (상세: 지퍼단추코드_규칙.txt)

지퍼: ZIPPER_사이즈코드
  ZIPPER_S / ZIPPER_M / ZIPPER_L

라벨코드 품목코드(4번째 자리)로 필요 부자재 자동 판단:
  T → PLASTIC 단추
  P → METAL 단추 또는 ZIPPER_S
  J → ZIPPER_M + METAL 단추
  D → ZIPPER_L

========================================
트렌드 추론 (이 회사 특이사항)
========================================

원목단추 출고량 월별 집계 → 플랫폼으로 트렌드 신호 전송
  WOOD 출고 +20% 이상 → "프리미엄 셔츠 수요 증가" 신호
  AI_로직.txt 트렌드 추론 섹션 참고

========================================
개발 시 주의사항
========================================

  - 품목별 기계가 다름 (목공/사출/프레스/조립) → 재고 테이블 item_name으로 구분
  - J(재킷) 주문 시 지퍼+단추 동시 계산 필요
  - UI 구조는 agent표준 구조/README.md에서 절대 벗어나지 말 것
  - 트렌드 신호 월 1회 플랫폼 전송 로직 구현
